# Rekrutacja — checklist procesu

_Od definiowania stanowiska do oferty przyjętej._

## Faza 0: Przygotowanie

### Zrozumienie potrzeby
- [ ] **Dlaczego rekrutujemy** — zastępstwo? wzrost? reorganizacja?
- [ ] **Budżet** — zakres płac + benefity + ZUS
- [ ] **Termin zatrudnienia** — kiedy ma zacząć
- [ ] **Kto decyduje** — hiring manager, zespół, zarząd

### Opis stanowiska (job description)
- [ ] **Tytuł stanowiska** (neutralny płciowo!)
- [ ] **Podległość** (komu raportuje)
- [ ] **Obowiązki główne** (5-8 punktów)
- [ ] **Kwalifikacje must-have** (wykształcenie, lata doświadczenia, umiejętności)
- [ ] **Kwalifikacje nice-to-have**
- [ ] **Widełki płacowe** — **OBOWIĄZKOWE W 2026** (dyrektywa jawności)

### Typ umowy
- [ ] Umowa o pracę (okres próbny → czas określony → nieokreślony)?
- [ ] Umowa zlecenie (rola nieregularna)?
- [ ] Umowa B2B (ekspert z własną firmą, wielu klientów)?
- [ ] Staż (dla studentów / absolwentów — do 18 mies.)

---

## Faza 1: Ogłoszenie

### Wymogi prawne (od 2026)
- [ ] **Widełki płacowe** lub min. stawka — obowiązkowo
- [ ] **Neutralny język** — bez "dla mężczyzn/kobiet"
- [ ] **Zakaz pytań o poprzednie wynagrodzenie** (nie w ogłoszeniu, nie na rozmowie)
- [ ] **Klauzula RODO** — administrator, cel, retencja, link do pełnej klauzuli

### Wymogi dobrej praktyki
- [ ] **Jasne obowiązki** — kandydat wie co ma robić
- [ ] **Realne wymagania** — nie "12 lat doświadczenia w technologii wymyślonej 8 lat temu"
- [ ] **Widełki z uzasadnieniem** — "w zależności od doświadczenia i kompetencji"
- [ ] **Benefity** — konkretnie, nie "atrakcyjny pakiet"
- [ ] **Sposób aplikacji** — mail? portal? platforma?
- [ ] **Termin zamknięcia naboru** — jeśli wpłynął
- [ ] **Pozytywne elementy** — kultura, rozwój, elastyczność

### Publikacja
- [ ] Strona firmowa / kariera
- [ ] LinkedIn
- [ ] Pracuj.pl / No Fluff Jobs / Just Join IT (IT)
- [ ] Portale branżowe
- [ ] Agencje rekrutacyjne (jeśli używamy)
- [ ] Networking / polecenia pracownicze

---

## Faza 2: Selekcja CV

### Filtr CV — kryteria
- [ ] **Must-have** — brak → odrzucamy (ale informujemy!)
- [ ] **Doświadczenie** — sprawdzenie dat, logiki kariery
- [ ] **Umiejętności** — konkretne technologie, narzędzia, języki
- [ ] **Miejsce zamieszkania** (jeśli stacjonarna praca)

### Co NIE bierzemy pod uwagę (RODO / antydyskryminacja)
- Wiek (jeśli nie jest bezpośrednim wymogiem)
- Płeć / tytuły grzecznościowe (pani/pan)
- Stan cywilny / plany rodzinne
- Wyznanie / orientacja / pochodzenie
- Zdjęcie (nie jest obowiązkowe)

### Feedback do kandydatów odrzuconych
- [ ] **Krótka, konkretna wiadomość** — "dziękujemy, nie kontynuujemy"
- [ ] **Opcjonalnie uzasadnienie** — jeśli kandydat pyta
- [ ] **Propozycja rekrutacji przyszłej** — jeśli zainteresowani
- [ ] **Nie zostawiamy bez odpowiedzi** — etyka, wizerunek firmy

### Archiwizacja
- [ ] CV do bazy (ATS) — tylko zaaplikowany cel
- [ ] Po rekrutacji (wybrany lub nie) — usunięcie po 6 mies., chyba że zgoda na przyszłe

---

## Faza 3: Rozmowa kwalifikacyjna

### Przygotowanie
- [ ] **Ustalenie osób** prowadzących (HR + hiring manager + ewentualnie kolega z zespołu)
- [ ] **Lista pytań** (przygotowana, nie improwizowana)
- [ ] **Case / zadanie** — jeśli stosowane (z wyprzedzeniem dla kandydata)
- [ ] **Miejsce / format** (stacjonarnie, zdalnie, Google Meet/Teams)

### Co pytamy (konkretnie, zgodnie z prawem)
- [ ] **Doświadczenie zawodowe** — konkretnie, projekty, sukcesy, porażki
- [ ] **Motywacja** — dlaczego my, dlaczego teraz
- [ ] **Oczekiwania** — rola, rozwój, warunki
- [ ] **Dostępność** — kiedy może zacząć
- [ ] **Kompetencje miękkie** — przykłady z życia (STAR: Situation, Task, Action, Result)
- [ ] **Case techniczny** — jeśli rola wymaga

### Czego NIE pytamy
- Wiek, stan cywilny, plany rodzinne
- **Poprzednie wynagrodzenie** (zakaz od 2026)
- Orientacja seksualna, wyznanie, poglądy polityczne
- Pytania, które nie wiążą się z kompetencjami do pracy
- "Czy zamierza Pani mieć dzieci" / "Ile Pani ma lat"

### Informacje dla kandydata
- [ ] **Krótka prezentacja firmy** i roli
- [ ] **Widełki** (jeszcze raz, konkretnie)
- [ ] **Benefity** (konkretnie, bez ogólników)
- [ ] **Dalsze kroki** (ile etapów, kiedy decyzja)
- [ ] **Czas na pytania** od kandydata

### Po rozmowie
- [ ] **Notatka w ATS** (lub Excel) — oceny, obserwacje, rekomendacja
- [ ] **Konsultacja w zespole rekrutacyjnym** (jeśli wiele osób)
- [ ] **Decyzja**: kolejny etap / odrzucamy / rezerwa

---

## Faza 4: Referencje / background check

### Referencje (tylko za zgodą kandydata)
- [ ] **Zgoda pisemna** kandydata na kontakt z referencjami
- [ ] **Lista referencji** podana przez kandydata (nie szukamy sami)
- [ ] **Rozmowa referencyjna** — poprzedni przełożony, kolega
- [ ] **Pytania referencyjne**:
  - Relacja pracownicza (jak długo, w jakiej roli)
  - Mocne strony
  - Obszary rozwoju
  - Powód odejścia (jeśli zna)
  - Czy by zatrudnił ponownie

### Weryfikacja dokumentów
- [ ] **Dyplomy** — w razie wątpliwości można zweryfikować na uczelni
- [ ] **Certyfikaty** — sprawdzenie aktualności online (jeśli jest baza)
- [ ] **Świadectwa pracy** — sprawdzenie dat

### Niekaralność (gdy wymagana)
- [ ] **Zaświadczenie z KRK** — kandydat dostarcza
- [ ] **Tylko dla stanowisk regulowanych** — nie żądamy "na wszelki wypadek"

---

## Faza 5: Oferta

### Przygotowanie oferty
- [ ] **Pisemna oferta** z:
  - Stanowiskiem
  - Wynagrodzeniem (brutto + benefity)
  - Typem umowy
  - Datą rozpoczęcia
  - Lokalizacją pracy (także hybryda)
  - Terminem do akceptacji (zwykle 3-5 dni)

### Negocjacje
- [ ] **Bądź gotowa** na kontrofertę
- [ ] **Znaj widełki** i margines negocjacyjny
- [ ] **Nie okłamuj** — jeśli coś nie jest możliwe, powiedz
- [ ] **Zapisuj uzgodnienia** — to wejdzie do umowy

### Akceptacja
- [ ] **Pisemne potwierdzenie** od kandydata
- [ ] **Przekazanie kandydata do fazy onboarding** (patrz `onboarding.md`)

### Odmowa kandydata
- [ ] **Jeśli odrzuca** — krótka rozmowa / mail, dlaczego
- [ ] **Zostaw otwarte drzwi** — jeśli dobry kandydat
- [ ] **Plan B** — kontynuacja z innym lub ponowna rekrutacja

## KPI rekrutacji (Janina monitoruje)

| KPI | Cel przykładowy |
|-----|-----------------|
| **Time-to-hire** | < 30 dni od ogłoszenia do podpisania oferty |
| **Time-to-offer** | < 14 dni od pierwszego kontaktu do oferty |
| **Offer acceptance rate** | > 80% |
| **Candidate experience** (NPS) | > 60 |
| **Cost per hire** | Zależy od roli |
| **Quality of hire** (rotacja w 1. roku) | < 15% |

---

## Typowe błędy Janiny przypomina

1. **Brak widełek w ogłoszeniu** — naruszenie dyrektywy 2026 + kara
2. **Pytania zakazane** — "Kiedy Pani planuje dziecko" — dyskryminacja
3. **Ghosting kandydatów** — brak feedbacku = zły wizerunek firmy
4. **Zbyt długi proces** — dobrzy kandydaci nie czekają
5. **Jednoosobowe decyzje** — 1 rekruter, 1 manager, brak drugiej opinii
6. **Rzutowanie bias** — "wyglądem do nas nie pasuje", "kobieta na magazyn"
7. **Brak dokumentacji decyzji** — po 3 miesiącach nikt nie pamięta dlaczego tak
8. **Zaniedbanie klauzuli RODO** — otrzymanie CV bez ujawnienia celu przetwarzania
